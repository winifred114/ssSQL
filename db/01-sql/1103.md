# SQLite
- 모바일 앱이나 소규모 프로그램에 적합
## DB
- Table은 데이터가 기록되는 곳
- 데이터는 기본 키 또는 외해 키를 통해 결합(join)될 수 있는 여러 테이블에 걸쳐 구조화 됨
## SQL
- 테이블의 형태로 구조화 된 관계형 데이터베이스에게 요청을 질의
- 도서관 사서 역할을 하는 DBMS에거 데이터를 찾아달라고 요청
### 쓰는 법
- SELECT, FROM 등 (SQL 키워드는 대소문자를 수분하지 않음 *대문자 권장)
- 각 SQL Statements 끝에는 세미콜론이 필요
 ### SQL Statements
 - SQL을 구성하는 가장 기본적인 코드 블록
 ### 수행 목적
 - CRUD
 - DDL (데이터 기본 구조 및 형식 변경)
 - DQL (데이터 검색)
 - DML (데이터 조작)
 - DCL (데이터 작업 및 사용자 권한 조작)
 ### 시작하기
 - Database Client 설치
 - Database -> SQLite -> path -> 파일 찾아서 ALL 선택
 ### DQL
 - FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY -> LIMIT
 #### SELECT FROM
 - SELECT 키워드 이후 데이터를 선택하려는 필드를 하나 이상 지정
 - FROM 키워드 이후 데이터를 선택하려는 테이블의 이름을 지정
 - **활용** 테이블 ***에서 000 필드의 모든 데이터를 조회 ( '*' 전체 데이터)
 - **활용** AS '00' 조회 결과에서만 이름 변경
 - **활용** 기본 연산도 가능 (Milliseconds / 60000 이면 시간(분) 가능)
#### ORDER BY
- FROM 뒤에 위치
- 하나 이상의 컬럼을 기준으로 결과를 오름차순(기본) 또는 내림차순 정렬 (ASC 오름, DESC 내림)(아스키코드 기준)
- ORDER BY를 두 번 이상 쓴다면 첫 번째 전체 정렬 이후 그 기준으로 오름차순(끊어질 수 있음)
- NULL 값이 있으면 오름차순 기준 가장 위
#### Filtering data
##### Clause
- DISTINCT 조회된 결과에서 중복된 데이터 제거 (두 개 이상이면 한 세트로 봐서 둘 중 하나만 중복 아니어도 통과)(SELECT 바로 다음)
- WHERE 조회 시 특정 검색 조건을 지정 (AND, OR 등의 연산자랑 많이 사용)(조건이 2개 이상이면 , 말고 연산자 사용)(FROM 다음 ORDER BY 이전)
- LIMIT a a개만 조회 a, b a~b까지만 조회 (ORDER BY 뒤에)
##### Operator
- BETWEEN a >= 10000 AND a <= 500000 대신 a BETWEEN 10000 AND 500000도 가능
- IN OR 여러개 처럼 사용 가능 IN ('canada', 'germany')
- LIKE '%' 0개 이상의 문자열과 일치하는지 확인, '_' 단일 문자와 일치하는지 확인
- Comparison
- LOgical
#### Grouping data
##### Aggregation Funtions
##### GROUP BY 레코드를 그룹화 하여 중복 제거 (FROM 뒤에)
- COUNT() 그룹된 레코드의 각 객체 개수
- AVG() 평균 
- HAVING GROUP BY에서 WHERE처럼 사용 그룹화 된 결과에서 WHERE 느낌
